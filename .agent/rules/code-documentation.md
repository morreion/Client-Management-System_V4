**Creating Code Explanation Markup files**  
- When doing an edit, update, change or add code , explain the code in the chat window, and create markup files for the "Code Explanations" done in chat after explaining the whole code explanation. Start with a 0-padded 3 digit number.
- The code examples in the markup .md file, if possible should follow the coloring theme of the IDE or code editor i'm using.
- Every line of the code, method and class should be properly explained in the markup file, what it does, what it means and how it relates to other code blocks or classes.
- The explanations should be properly ordered and if possible make a table of contents at the top of the file of the subjects , concepts or type of code explained
- The code explanation files should be stored in the Applications directory in a special folder named Code Explanations.
