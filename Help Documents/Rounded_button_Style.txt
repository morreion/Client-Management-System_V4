You should create a new style inside that same file. 
The existing styles you posted are very specific (one is for Sidebar RadioButtons, 
the others are for Window Icons like Close/Minimize).

To fulfill your requirement of customizable hover colors and rounded corners, 
the best trick in WPF without writing complex C# backend code is to use the 
Tag property of the button to hold your "Hover Color".

Here is the plan:

Rounded Corners: We will use a Border with CornerRadius inside the template.

Hover Color: We will tell the template: "When the mouse is over the button, 
switch the Background to whatever color is stored in the Tag property."

Step 1: Add this Style to your Resource Dictionary

Open your existing file (e.g., ButtonStyles.xaml or whatever you named the file 
containing the code you pasted) and add this new style at the bottom, 
before the closing </ResourceDictionary> tag.


<!-- NEW: Rounded Action Button Style -->
    <!-- Usage: Set Background for normal color, Set Tag for Hover color -->
    <Style x:Key="RoundedButtonStyle" TargetType="Button">
        <Setter Property="Foreground" Value="White"/>
        <Setter Property="Padding" Value="15,8"/>
        <Setter Property="Margin" Value="0,0,10,0"/>
        <Setter Property="BorderThickness" Value="0"/>
        <Setter Property="FontSize" Value="14"/>
        <Setter Property="FontWeight" Value="SemiBold"/>
        <Setter Property="Cursor" Value="Hand"/>
        
        <Setter Property="Template">
            <Setter.Value>
                <ControlTemplate TargetType="Button">
                    <Border x:Name="border"
                            Background="{TemplateBinding Background}"
                            CornerRadius="8" 
                            BorderThickness="{TemplateBinding BorderThickness}"
                            BorderBrush="{TemplateBinding BorderBrush}">
                        
                        <ContentPresenter HorizontalAlignment="Center" 
                                          VerticalAlignment="Center"/>
                    </Border>
                    
                    <ControlTemplate.Triggers>
                        <!-- TRIGGER: Mouse Over -->
                        <!-- This changes the background to the color defined in the Tag property -->
                        <Trigger Property="IsMouseOver" Value="True">
                            <Setter TargetName="border" Property="Background" 
                                    Value="{Binding RelativeSource={RelativeSource TemplatedParent}, Path=Tag}"/> 
                        </Trigger>
                        
                        <!-- TRIGGER: Button Pressed -->
                        <Trigger Property="IsPressed" Value="True">
                            <Setter TargetName="border" Property="Opacity" Value="0.8"/>
                        </Trigger>
                        
                        <!-- TRIGGER: Disabled -->
                        <Trigger Property="IsEnabled" Value="False">
                            <Setter TargetName="border" Property="Background" Value="#555555"/>
                            <Setter TargetName="border" Property="Opacity" Value="0.5"/>
                            <Setter Property="Foreground" Value="#AAAAAA"/>
                        </Trigger>
                    </ControlTemplate.Triggers>
                </ControlTemplate>
            </Setter.Value>
        </Setter>
    </Style>

Explanation of the Solution

Why a new style? The existing BtnStyle in your file targeted RadioButton. 
You needed a style for Button.

CornerRadius: CornerRadius is not a property of a Button, 
but it is a property of a Border. By putting a Border inside the ControlTemplate of the button,
we can curve the edges.

The Tag Trick: You asked to be able to set the hover color easily. 
By binding the IsMouseOver trigger to the Tag property, 
you can set the hover color directly in your XAML (e.g., Tag="Red") 
without needing to write a new style for every single color button you have.

Is there a different way instead of adding space to the buttons 
content property?



The reason your Padding property is being ignored is that the ControlTemplate 
created in the previous step wasn't told to use it. In WPF, 
if you don't explicitly bind the ContentPresenter to the Padding of the button, 
the text will just sit tightly in the center, ignoring whatever values you set.

You do not need to add spaces to the text. Here is the fix.


Step 1: Fix the Style in ButtonStyles.xaml

Go to your RoundedButtonStyle in your resource file. 
Look for the <ContentPresenter /> tag inside the Border.

You need to add Margin="{TemplateBinding Padding}".

Replace the entire ControlTemplate section of that style with this corrected version:


<Setter Property="Template">
    <Setter.Value>
        <ControlTemplate TargetType="Button">
            <Border x:Name="border"
                    Background="{TemplateBinding Background}"
                    CornerRadius="8" 
                    BorderThickness="{TemplateBinding BorderThickness}"
                    BorderBrush="{TemplateBinding BorderBrush}">
                
                <!-- THE FIX: Add Margin="{TemplateBinding Padding}" -->
                <!-- This tells the text to respect the padding set on the Button -->
                <ContentPresenter HorizontalAlignment="Center" 
                                  VerticalAlignment="Center"
                                  Margin="{TemplateBinding Padding}"/>
            </Border>
            
            <ControlTemplate.Triggers>
                <Trigger Property="IsMouseOver" Value="True">
                    <Setter TargetName="border" Property="Background" 
                            Value="{Binding RelativeSource={RelativeSource TemplatedParent}, Path=Tag}"/> 
                </Trigger>
                <Trigger Property="IsPressed" Value="True">
                    <Setter TargetName="border" Property="Opacity" Value="0.8"/>
                </Trigger>
                <Trigger Property="IsEnabled" Value="False">
                    <Setter TargetName="border" Property="Background" Value="#555555"/>
                    <Setter TargetName="border" Property="Opacity" Value="0.5"/>
                    <Setter Property="Foreground" Value="#AAAAAA"/>
                </Trigger>
            </ControlTemplate.Triggers>
        </ControlTemplate>
    </Setter.Value>
</Setter>

is it possible within the style file --> ButtonStyles.xaml, 
to implement a tooltip for the button?



Yes, it is absolutely possible. In WPF, the ToolTip is actually 
its own control, separate from the Button. 
To make it look good (Dark theme, rounded corners), 
you need to create a Style for the ToolTip inside your 
ButtonStyles.xaml file.

Once you add this style, any button (or control) that has 
a tooltip will automatically use this custom dark design.

Step 1: Add the ToolTip Style to ButtonStyles.xaml

Open your ButtonStyles.xaml file and paste this code above your 
RoundedButtonStyle.

Key Features of this Style:

TargetType="ToolTip": This automatically applies to all tooltips.

Rounded Corners: Uses a Border with CornerRadius="8".

Dark Theme: Matches your app colors (#2D3035 background, #3F4347 border).

DropShadow: Adds a subtle shadow so it "pops" out from the background.


<!-- CUSTOM TOOLTIP STYLE -->
    <Style TargetType="ToolTip">
        <Setter Property="Background" Value="#2D3035"/>
        <Setter Property="Foreground" Value="#DBDBDB"/>
        <Setter Property="BorderBrush" Value="#3F4347"/>
        <Setter Property="BorderThickness" Value="1"/>
        <Setter Property="FontSize" Value="12"/>
        <Setter Property="Padding" Value="10,5"/>
        <!-- Optional: offset slightly from the mouse cursor -->
        <Setter Property="HorizontalOffset" Value="10"/> 
        <Setter Property="VerticalOffset" Value="10"/>
        
        <Setter Property="Template">
            <Setter.Value>
                <ControlTemplate TargetType="ToolTip">
                    <Grid>
                        <!-- Shadow Effect -->
                        <Border Background="Black" CornerRadius="8" Margin="3" Opacity="0.3">
                            <Border.Effect>
                                <BlurEffect Radius="5"/>
                            </Border.Effect>
                        </Border>

                        <!-- Actual ToolTip Box -->
                        <Border Background="{TemplateBinding Background}"
                                BorderBrush="{TemplateBinding BorderBrush}"
                                BorderThickness="{TemplateBinding BorderThickness}"
                                CornerRadius="8"
                                Padding="{TemplateBinding Padding}">
                            <ContentPresenter HorizontalAlignment="Center" 
                                              VerticalAlignment="Center"/>
                        </Border>
                    </Grid>
                </ControlTemplate>
            </Setter.Value>
        </Setter>
    </Style>




Step 2: How to use it in your View (Treatment.xaml)

You do not need to change the Button Style (RoundedButtonStyle). You simply add the ToolTip property to your buttons in the view.

Example for your Save and Add buttons:


<!-- Save Button -->
<Button Content="ðŸ’¾ Save" 
        Command="{Binding SaveCommand}"
        Style="{StaticResource RoundedButtonStyle}"
        Background="#3B82F6" 
        Tag="#2563EB"
        IsEnabled="{Binding IsSelectionActive}"
        ToolTip="Save the current treatment details to the database" /> 

<!-- Add Button -->
<Button Content="âž• Add New" 
        Command="{Binding AddCommand}" 
        Style="{StaticResource RoundedButtonStyle}"  
        Background="#10B981" 
        Tag="#059669"
        ToolTip="Create a new blank treatment record" />

Now, when you hover over these buttons, a dark, rounded popup will appear with the text you wrote in the ToolTip property.
