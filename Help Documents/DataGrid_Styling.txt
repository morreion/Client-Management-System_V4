            The GridLinesVisibility property accepts one of four enum values. To make them visible, you usually set it to All.

Here is the breakdown of your questions and the solutions for customizing thickness and color, which require slightly different approaches for the main grid versus the headers.

1. Grid Lines Visibility, Color, and Thickness

Standard grid lines in WPF do not have a direct "Thickness" property. To change the thickness, the standard workaround is to turn off the default grid lines and use a CellStyle to draw borders on the cells instead.

Visibility: Set GridLinesVisibility="All" (or Horizontal/Vertical).[1]

Color: Use HorizontalGridLinesBrush and VerticalGridLinesBrush.[2][3]

Thickness: Use a CellStyle (recommended method).

Updated DataGrid XAML:

<DataGrid x:Name="ClientsDataGrid" 
          GridLinesVisibility="All" 
          HorizontalGridLinesBrush="#505050" 
          VerticalGridLinesBrush="#505050" 
          ...>
    <!-- Rest of your properties -->
</DataGrid>

If you need a specific Thickness (e.g., 2px):
Set GridLinesVisibility="None" and add this to your DataGrid's resources or CellStyle:

<DataGrid.CellStyle>
    <Style TargetType="DataGridCell">
        <!-- Set the border thickness (Right and Bottom effectively create the grid) -->
        <Setter Property="BorderThickness" Value="0,0,1,1"/>
        <Setter Property="BorderBrush" Value="#505050"/>
        <!-- Optional: Fix for content overlapping the border -->
        <Setter Property="Template">
            <Setter.Value>
                <ControlTemplate TargetType="DataGridCell">
                    <Border BorderBrush="{TemplateBinding BorderBrush}" 
                            BorderThickness="{TemplateBinding BorderThickness}" 
                            Background="{TemplateBinding Background}" 
                            SnapsToDevicePixels="True">
                        <ContentPresenter SnapsToDevicePixels="{TemplateBinding SnapsToDevicePixels}"/>
                    </Border>
                </ControlTemplate>
            </Setter.Value>
        </Setter>
    </Style>
</DataGrid.CellStyle>
2. Column Header Separators

The column separators are part of the DataGridColumnHeader style.[4][5]

Visibility: Controlled by the SeparatorVisibility property.[4][5]

Color: Controlled by the SeparatorBrush property.

Thickness: There is no direct property for separator thickness. The separator is built into the default control template. To change its thickness, you must override the Template of the header.

Easy Fix (Color & Visibility only):
Update your existing DataGrid.ColumnHeaderStyle with these setters:

code
Xml

expand_less
<DataGrid.ColumnHeaderStyle>
    <Style TargetType="DataGridColumnHeader">
        <Setter Property="Background" Value="#3F4347"/>
        <Setter Property="Foreground" Value="#DBDBDB"/>
        <Setter Property="Padding" Value="10,5"/>
        <Setter Property="FontWeight" Value="SemiBold"/>
        <Setter Property="BorderThickness" Value="0"/>
        
        <!-- VISIBILITY: "Visible", "Hidden", or "Collapsed" -->
        <Setter Property="SeparatorVisibility" Value="Visible"/>
        
        <!-- COLOR -->
        <Setter Property="SeparatorBrush" Value="Red"/>
    </Style>
</DataGrid.ColumnHeaderStyle>

Advanced Fix (Changing Thickness):
Since you cannot simply set "SeparatorThickness," you can simulate it by adding a right-side border to your header style.

Set SeparatorVisibility to Collapsed (to hide the default thin line).

Add a BorderThickness of 0,0,2,0 (Right side only) to the Header itself.

Set the BorderBrush to your desired separator color.

        <Setter Property="FontWeight" Value="SemiBold"/>
        
        <!-- Hide default separator -->
        <Setter Property="SeparatorVisibility" Value="Collapsed"/>

******************Hoverover and selected rows styles******************

I have a datagrid and want to have an effect. When i hover over 
the rows with a mouse, the row that is hovered over should 
change color, as well as the text in that column, when the mouse 
leaves the row should revert back to its normal properties. These 
same hover over properties for the row, should also applied when a 
row is selected, so same foregound and background colors as when the 
mouse hovers over it. Also when hovering over the rows, or selecting 
the row, the cells border of that row should change color, for instance 
light blue.This style should apply to all the datagrids in all the views. 
How do i go about that ? Use the view and its datagrid i provided before as the context




To achieve this globally for all DataGrids, we will follow the 
same pattern as before: create a dedicated ResourceDictionary file.

Because you want the Cell Borders to change color when the 
Row is hovered or selected, we actually need to style two 
things:

DataGridRow: To handle the background and text color changes.

DataGridCell: To handle the light blue border effect (since rows 
don't actually have "vertical" borders between columns, cells do).



Step 1: Create the Style File

Go to your Styles folder.

Add a new Resource Dictionary named DataGridStyles.xaml.


Step 2: Add the Code

Paste the following code into DataGridStyles.xaml.

Key Logic:

DataGridCell Style: We use DataTrigger to look up at the parent DataGridRow. If the Row is hovered or selected, the Cell changes its BorderBrush to Light Blue.

DataGridRow Style: We use standard Trigger properties. 
If IsMouseOver or IsSelected is true, we change the Background 
to a lighter gray and Foreground to White.



<ResourceDictionary xmlns="http://schemas.microsoft.com/winfx/2006/xaml/presentation"
                    xmlns:x="http://schemas.microsoft.com/winfx/2006/xaml">

    <!-- 1. The Cell Style (Handles the Light Blue Borders) -->
    <Style x:Key="GlobalDataGridCellStyle" TargetType="DataGridCell">
        <!-- Default State: Transparent border (invisible), dark background -->
        <Setter Property="Background" Value="Transparent"/>
        <Setter Property="BorderBrush" Value="Transparent"/>
        <Setter Property="BorderThickness" Value="0,0,1,1"/> <!-- Right and Bottom borders -->
        <Setter Property="Padding" Value="5"/>
        <Setter Property="Template">
            <Setter.Value>
                <ControlTemplate TargetType="DataGridCell">
                    <Border BorderBrush="{TemplateBinding BorderBrush}" 
                            BorderThickness="{TemplateBinding BorderThickness}" 
                            Background="{TemplateBinding Background}" 
                            SnapsToDevicePixels="True">
                        <ContentPresenter SnapsToDevicePixels="{TemplateBinding SnapsToDevicePixels}"
                                          VerticalAlignment="Center"
                                          Margin="{TemplateBinding Padding}"/>
                    </Border>
                </ControlTemplate>
            </Setter.Value>
        </Setter>

        <Style.Triggers>
            <!-- TRIGGER: When Parent ROW is Hovered -->
            <DataTrigger Binding="{Binding RelativeSource={RelativeSource AncestorType=DataGridRow}, Path=IsMouseOver}" Value="True">
                <Setter Property="BorderBrush" Value="#3B82F6"/> <!-- Light Blue Border -->
            </DataTrigger>

            <!-- TRIGGER: When Parent ROW is Selected -->
            <DataTrigger Binding="{Binding RelativeSource={RelativeSource AncestorType=DataGridRow}, Path=IsSelected}" Value="True">
                <Setter Property="BorderBrush" Value="#3B82F6"/> <!-- Light Blue Border -->
                <Setter Property="Foreground" Value="White"/>
            </DataTrigger>
            
            <!-- Remove the default blue selection highlight on the specific cell, 
                 so the whole Row looks uniform -->
            <Trigger Property="IsSelected" Value="True">
                <Setter Property="Background" Value="Transparent"/> 
                <Setter Property="Foreground" Value="White"/>
            </Trigger>
        </Style.Triggers>
    </Style>

    <!-- 2. The Row Style (Handles Background and Text Color) -->
    <Style x:Key="GlobalDataGridRowStyle" TargetType="DataGridRow">
        <Setter Property="Background" Value="#2D3035"/> <!-- Normal Background -->
        <Setter Property="Foreground" Value="#DBDBDB"/> <!-- Normal Text -->
        <Setter Property="BorderThickness" Value="0"/>
        <Setter Property="FontSize" Value="14"/>
        <Setter Property="Padding" Value="0,5"/>
        
        <Style.Triggers>
            <!-- Mouse Hover Effect -->
            <Trigger Property="IsMouseOver" Value="True">
                <Setter Property="Background" Value="#4B5563"/> <!-- Lighter Grey on Hover -->
                <Setter Property="Foreground" Value="White"/>    <!-- White Text on Hover -->
            </Trigger>

            <!-- Selection Effect (Same as Hover) -->
            <Trigger Property="IsSelected" Value="True">
                <Setter Property="Background" Value="#4B5563"/> <!-- Lighter Grey on Select -->
                <Setter Property="Foreground" Value="White"/>    <!-- White Text on Select -->
            </Trigger>
        </Style.Triggers>
    </Style>

</ResourceDictionary>



Step 3: Register in App.xaml

Update your App.xaml to include this new dictionary, 
just like you did for the textboxes.



<Application.Resources>
    <ResourceDictionary>
        <ResourceDictionary.MergedDictionaries>
            <!-- Your other styles -->
            <ResourceDictionary Source="Styles/TextBoxStyles.xaml"/>
            
            <!-- ADD THIS LINE: -->
            <ResourceDictionary Source="Styles/DataGridStyles.xaml"/>
            
        </ResourceDictionary.MergedDictionaries>
    </ResourceDictionary>
</Application.Resources>

Created DataGridStyles.xaml: Contains the logic to turn 
rows/cells Gray+White on hover, and add Blue borders.

Cleaned Treatment.xaml: Removed specific color properties

Style triggers are allowed to activate.

Applied Styles: Added RowStyle="{StaticResource 
GlobalDataGridRowStyle}" and CellStyle="{StaticResource 
GlobalDataGridCellStyle}" to the DataGrid.
